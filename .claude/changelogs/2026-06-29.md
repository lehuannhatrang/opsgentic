# 2026-06-29

## One-command demo setup

- Added `hack/demo-up.sh` — interactive, idempotent orchestrator that stands up the full
  end-to-end demo: detects/creates a cluster (current context, or local k3d/minikube/kind),
  reads a GitHub fine-grained PAT and auto-forks `lehuannhatrang/demo-workload`, prompts for an
  optional LLM endpoint, installs ArgoCD and kube-prometheus-stack via Helm (Alertmanager
  selectors pre-wired so step B5 is no longer manual), creates one ArgoCD Application per
  `apps/*` pointing at the user's fork, generates `bootstrap.env`, and runs `./bootstrap.sh`.
  Prints access URLs and a verification `curl` at the end. Most prompts are overridable via env
  vars.
- Added `hack/demo-down.sh` — tears down the ArgoCD Applications, OpsGentic, and the Helm
  releases. Deliberately does NOT delete the cluster; only prints the delete command for the
  user to run.
- Updated `QUICKSTART.md` — added section "0. One-command demo" as the recommended fastest path,
  ahead of the existing local-dev (A) and manual Kubernetes (B) paths.
- Updated `README.md` — Quick start now leads with the one-command demo.

- `hack/demo-up.sh`: `CLUSTER_CHOICE` now also accepts `k8s` (current context) and lists it in
  the interactive prompt; `CLUSTER_CHOICE=k8s` with an unreachable context fails fast.

- `hack/demo-up.sh` fixes: color vars now use ANSI-C quoting (`$'\033...'`) so escape codes
  render correctly inside heredocs (the GitHub link was printing literal `\033[1m`); call
  `bootstrap.sh bootstrap.env` with a basename, since bootstrap.sh prefixes `./` to its arg and
  an absolute path broke sourcing.

Notes: `bootstrap.env` (contains the PAT) was already gitignored. No commits were made.

# 2026-06-20

## M1 — Runnable skeleton (MVP scaffold)

Bootstrapped `opsgentic`: a multi-agent RCA / Validation / Action system on LangGraph with GitOps human-in-the-loop. The LLM uses a local vLLM (OpenAI-compatible) endpoint via `langchain_openai.ChatOpenAI`, configured entirely through environment variables.

### Added
- `pyproject.toml`, `Dockerfile`, `.env.example`; updated `.gitignore` (ignore `.env`, venv, secrets manifest).
- `src/opsgentic/config.py` — Settings (pydantic-settings); LLM/GitOps/graph config via env.
- `src/opsgentic/agents/llm.py` — `ChatOpenAI` factory pointing at vLLM; returns `None` when unconfigured (enables the canned fallback).
- `src/opsgentic/graph/state.py` — `MachineState` (TypedDict) + `RemediationPlan`.
- `src/opsgentic/graph/builder.py` — `StateGraph` RCA→Validation→Action; conditional edge (self-heal loop capped by `max_rca_attempts`); `interrupt_before=["action"]`.
- `src/opsgentic/graph/nodes/{rca,validation,action}.py` — the three core nodes (real LLM path or canned fallback).
- `src/opsgentic/skills/{registry,metric_checks}.py` — Validation Skills (plain Python, independent of MCP).
- `src/opsgentic/triggers/normalize.py` — normalizes the two triggers: Grafana webhook + user chat.
- `src/opsgentic/gitops/pr.py` — PR creation stub (real implementation in M3).
- `src/opsgentic/runner.py` — app singleton + `MemorySaver`; `start_run` / `approve` / `reject` / `snapshot`.
- `src/opsgentic/main.py` — FastAPI: `/webhook/grafana`, `/chat`, `/runs/{id}`, `/runs/{id}/approve|reject`, `/healthz`.
- `src/opsgentic/cli.py` — local runner.
- `examples/{grafana_alert,chat_input}.json`.

### Tested
- `py_compile` over all of `src`: OK.
- End-to-end CLI run (grafana, `--approve`): RCA → Validation → pause before `action` (`awaiting_approval=true`) → approve → `action` opens a stub PR → `execution_status=applied`. Confirms `interrupt_before` behaves correctly.
- Chat trigger: normalization and run up to the approval pause: OK.
